# Glossary
## CS Learning Platform - BGU

### A
- **Anonymization:** Process ensuring teacher dashboard cannot link conversations to individual students
- **API Rate Limiting:** Max 10 chatbot questions per student per day to control costs

### B
- **Base Case:** Termination condition in recursion that prevents infinite loops
- **BGU:** Ben Gurion University, located in Beer Sheva, southern Israel
- **Bloom's Taxonomy:** Classification of learning objectives (Remember, Understand, Apply, Analyze)

### C
- **CIS Faculty:** Computer and Information Sciences Faculty at BGU
- **Conversational Depth:** Metric measuring conceptual vs. syntax questions in chat sessions

### D  
- **DSPy:** Framework for structured LLM prompting, used for Socratic chatbot implementation

### L
- **Learning Trajectory:** Ordered sequence of topics with prerequisites (e.g., Variables → Loops → Functions)
- **LLM-Resilient Assignment:** Assignment designed to resist ChatGPT cheating via variations

### M
- **Mastery Threshold:** Score of ≥80% on post-quiz indicating topic is mastered

### P
- **Post-Quiz:** Assessment after chatbot session measuring learning gained
- **Pre-Quiz:** Baseline assessment before chatbot interaction  

### R
- **RAG (Retrieval-Augmented Generation):** Using vector DB to retrieve course materials as context for chatbot
- **Recursion:** Function calling itself - key intro CS topic, often confusing for students

### S
- **Socratic Method:** Teaching by asking questions rather than giving answers directly

### V
- **Vector Database:** Stores course material embeddings for semantic search (Pinecone/Weaviate)
